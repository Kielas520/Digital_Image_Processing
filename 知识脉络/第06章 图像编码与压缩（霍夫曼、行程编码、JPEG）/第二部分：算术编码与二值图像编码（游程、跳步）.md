## 一、 算术编码 (Arithmetic Coding)

霍夫曼编码需要为每个符号分配整数个比特，无法逼近非整数比特的理想熵极值。**算术编码是直接将一整串信息流（消息）编码为实数区间 $[0.0, 1.0)$ 内的一个唯一的小数 $n$。**

### 1. 严格数学递推公式（区间收缩）
设要编码的信息流（消息）为一串字符序列，各个信源符号在 $[0, 1)$ 中拥有自己初始分配的累积概率区间 $[l_i, h_i)$。
我们定义当前的累积编码区间为 $[L(k), H(k))$，其中初始区间 $[L(0), H(0)) = [0.0, 1.0)$。

当读入第 $k$ 个符号时，累积区间按照以下**区间递推迭代公式**进行收缩：
$$ L(k) = L(k-1) + [ H(k-1) - L(k-1) ] \times l_k $$
$$ H(k) = L(k-1) + [ H(k-1) - L(k-1) ] \times h_k $$
*变量含义：*
- $[L(k-1), H(k-1))$：前 $k-1$ 个字符计算出的累计区间。
- $H(k-1) - L(k-1)$：当前区间的宽度。
- $[l_k, h_k)$：当前输入的这第 $k$ 个字符对应的初始概率区间。

---

### 2. 算术编码手把手迭代例题（以消息 "bcadc" 为例）
设信源符号及其初始概率、累积区间划分如下表：
- `a`: $P(a) = 0.2 \rightarrow$ 初始区间为 $[0.0, 0.2)$，长度 0.2
- `b`: $P(b) = 0.3 \rightarrow$ 初始区间为 $[0.2, 0.5)$，长度 0.3
- `c`: $P(c) = 0.4 \rightarrow$ 初始区间为 $[0.5, 0.9)$，长度 0.4
- `d`: $P(d) = 0.1 \rightarrow$ 初始区间为 $[0.9, 1.0)$，长度 0.1

#### 【区间迭代过程】
1. **初始状态**：$[L(0), H(0)) = [0.0, 1.0)$，宽度 $W=1.0$。
2. **读入第1个字符 `b`** ($[l, h) = [0.2, 0.5)$)：
   - $L(1) = 0.0 + 1.0 \times 0.2 = 0.2$
   - $H(1) = 0.0 + 1.0 \times 0.5 = 0.5$
   - *当前区间*：$[0.2, 0.5)$，宽度 $W = 0.3$。
3. **读入第2个字符 `c`** ($[l, h) = [0.5, 0.9)$)：
   - $L(2) = 0.2 + 0.3 \times 0.5 = 0.35$
   - $H(2) = 0.2 + 0.3 \times 0.9 = 0.47$
   - *当前区间*：$[0.35, 0.47)$，宽度 $W = 0.12$。
4. **读入第3个字符 `a`** ($[l, h) = [0.0, 0.2)$)：
   - $L(3) = 0.35 + 0.12 \times 0.0 = 0.35$
   - $H(3) = 0.35 + 0.12 \times 0.2 = 0.374$
   - *当前区间*：$[0.35, 0.374)$，宽度 $W = 0.024$。
5. **读入第4个字符 `d`** ($[l, h) = [0.9, 1.0)$)：
   - $L(4) = 0.35 + 0.024 \times 0.9 = 0.3716$
   - $H(4) = 0.35 + 0.024 \times 1.0 = 0.374$
   - *当前区间*：$[0.3716, 0.374)$，宽度 $W = 0.0024$。
6. **读入第5个字符 `c`** ($[l, h) = [0.5, 0.9)$)：
   - $L(5) = 0.3716 + 0.0024 \times 0.5 = 0.3728$
   - $H(5) = 0.3716 + 0.0024 \times 0.9 = 0.37376$
   - *当前区间*：$[0.3728, 0.37376)$。
- **输出编码值**：在该区间内挑取一个最简的实数作为算术编码值（例如选 **$0.373$** 并转换为二进制）。

---

### 3. 算术译码（解码）标准步骤
要成功解码，必须事先知道**待解码的消息长度**。
- **解码步骤**：
  1. 拿接收到的编码值 $V$（如 $V = 0.25$）与初始符号区间对比，看它落在哪个区间，即输出该区间的符号。
  2. 根据输出符号，将当前区间 $[L, H)$ 进行逆向还原伸展，得到新的归一化对比值。
  3. 重复判定直到解码出了预定长度的所有字符。

## 二、 二值图像专用编码方法

二值图像（文档、图纸、表格）中只有前景色（黑 `1`）和背景色（白 `0`），图像具有大面积完全连续相同的特点。

### 1. 跳跃空白编码 (Skip Blank Coding, SBC)
- **步骤原理**：
  1. 将图像的每一条水平扫描线切分成若干等长的段，每段包含 $m$ 个像素（通常 $m = 8 \sim 12$）。
  2. 分类进行编码：
     - **第一类（全是 0，空白块）**：直接用 **1位码字 `0`** 来代表这一整段。
     - **第二类（含 1，非空白块）**：用 **码字 `1` 开头**，后面直接跟上这 $m$ 个像素的真实二值序列。
- **物理特性**：在含有大量留白背景的文档图像中，能以极高效率抹除背景冗余。

### 2. 游程长度编码 (Run-Length Coding, RLC)
- **物理定义**：
  游程编码记录的是**相同灰度像素连续出现的个数（游程长度）**。由于二值图像每行都是黑白段交替，我们只需记录每次变色前连续数字的个数。

- **定长与变长游程编码的区别**：
  - **定长游程编码**：记录游程长度的二进制位数是**完全固定**的（比如规定用 3 位表示，最大能表示长度为 7 的连续段。一旦连续相同的像素超过 7 个，就强行截断，转入下一轮游程编码）。
  - **变长游程编码**：根据游程发生的统计规律，对不同范围的游程使用**不同的位数**来编码（如频繁出现的短游程用短位数，不常用的长游程用长位数，这正是著名的 **MH / MR 传真压缩编码标准** 的核心）。
- **计算例题**：对于二值序列 `0000001111100011001`：
  - 默认规定从符号 `0` 开始。
  - 连续 `0` 有 6个 $\rightarrow$ 游程 6；
  - 连续 `1` 有 5个 $\rightarrow$ 游程 5；
  - 连续 `0` 有 3个 $\rightarrow$ 游程 3；
  - 连续 `1` 有 2个 $\rightarrow$ 游程 2；
  - 连续 `0` 有 2个 $\rightarrow$ 游程 2；
  - 连续 `1` 有 1个 $\rightarrow$ 游程 1；
  - **最终游程编码序列**：`6, 5, 3, 2, 2, 1`。只要利用该序列就能 100% 无损复原出原始二进制流。
